Policy Definitions

Define and manage Azure Policy definitions as Kubernetes custom resources using AzurePolicyDefinition.

4 min read

The AzurePolicyDefinition custom resource maps directly to an Azure Policy Definition. The operator reconciles each resource by creating or updating the corresponding definition in Azure.

API version for this resource. Must be policy.azure.com/v1alpha1.

apiVersion: policy.azure.com/v1alpha1

Resource kind. Must be AzurePolicyDefinition.

kind: AzurePolicyDefinition

Standard Kubernetes object metadata.

Unique name of the resource within its namespace.

metadata:
  name: require-https-storage

Kubernetes namespace.

metadata:
  namespace: platform-policies

Map of key/value pairs for organizing resources.

metadata:
  labels:
    team: platform
    env: production

Arbitrary non-identifying metadata.

metadata:
  annotations:
    policy.azure.com/owner: platform-team

Desired state of the AzurePolicyDefinition.

Human-readable name shown in the Azure portal and in compliance reports.

spec:
  displayName: "Require HTTPS on Storage Accounts"

A human-readable explanation of what the policy enforces and why.

spec:
  description: "Audits storage accounts that do not enforce HTTPS-only traffic."

Classifies the policy definition.

Allowed values: BuiltIn Custom NotSpecified Static

Default: Custom

spec:
  policyType: Custom

Controls which resource types the policy evaluates. All includes resource groups and subscriptions; Indexed targets only types that support tags and location.

Allowed values: All Indexed 

spec:
  mode: Indexed   # tag-aware resources only
  # mode: All     # includes resource groups and subscriptions

Semantic version of the policy definition (e.g. 1.0.0). Written into Azure Policy metadata under the version key; takes precedence over any version key in spec.metadata.

spec:
  version: "1.0.0"

Arbitrary metadata attached to the Azure Policy definition as a raw JSON object. Common keys: category, version.

spec:
  metadata:
    category: "Storage"
    version: "1.0.0"

Declares the parameters that the policy rule can reference. Each key is a parameter name following the Azure Policy parameter definition schema.

parameters:
  effect:
    type: String
    metadata:
      displayName: Effect
    allowedValues: [Audit, Deny, Disabled]
    defaultValue: Audit

The policy logic as an inline object. Exactly one of policyRule or policyRuleJson must be specified.

Mutually exclusive with policyRuleJson.

Condition that evaluates the resource. Supports field, allOf, anyOf, not, and count expressions.

spec:
  policyRule:
    if:
      allOf:
        - field: "type"
          equals: "Microsoft.Storage/storageAccounts"
        - field: "Microsoft.Storage/storageAccounts/supportsHttpsTrafficOnly"
          notEquals: true

Effect to apply when the condition is met (e.g. Audit, Deny, Modify, DeployIfNotExists).

spec:
  policyRule:
    then:
      effect: "[parameters('effect')]"

The policy rule as a raw JSON string. Use this when importing large pre-built definitions where inline YAML is impractical. Exactly one of policyRule or policyRuleJson must be specified.

Mutually exclusive with policyRule.

spec:
  policyRuleJson: |
    {
      "if": {
        "allOf": [
          { "field": "type", "equals": "Microsoft.Compute/virtualMachines" },
          { "field": "location", "notIn": ["australiaeast", "australiasoutheast"] }
        ]
      },
      "then": { "effect": "Deny" }
    }

Azure Subscription ID to create the policy definition in. When omitted, the definition is created at management group scope (requires managementGroupId).

spec:
  subscriptionId: "00000000-0000-0000-0000-000000000000"

Management Group ID to create the policy definition in. Use this to share definitions across multiple subscriptions.

spec:
  managementGroupId: "my-management-group"

Audit storage accounts without HTTPS

apiVersion: policy.azure.com/v1alpha1
kind: AzurePolicyDefinition
metadata:
  name: require-https-storage
spec:
  displayName: "Require HTTPS on Storage Accounts"
  description: "Audits storage accounts that do not enforce HTTPS-only traffic."
  policyType: Custom
  mode: Indexed
  version: "1.0.0"
  metadata:
    category: "Storage"
  parameters:
    effect:
      type: String
      metadata:
        displayName: Effect
      allowedValues: [Audit, Deny, Disabled]
      defaultValue: Audit
  policyRule:
    if:
      allOf:
        - field: "type"
          equals: "Microsoft.Storage/storageAccounts"
        - field: "Microsoft.Storage/storageAccounts/supportsHttpsTrafficOnly"
          notEquals: true
    then:
      effect: "[parameters('effect')]"
  subscriptionId: "00000000-0000-0000-0000-000000000000"

Deny untagged resources at management group scope

apiVersion: policy.azure.com/v1alpha1
kind: AzurePolicyDefinition
metadata:
  name: require-cost-center-tag
spec:
  displayName: "Require CostCenter tag on resources"
  description: "Denies creation of resources missing the CostCenter tag."
  policyType: Custom
  mode: Indexed
  version: "2.0.1"
  metadata:
    category: "Tags"
  policyRule:
    if:
      field: "tags['CostCenter']"
      exists: false
    then:
      effect: "Deny"
  managementGroupId: "my-management-group"

Import a large policy from raw JSON

apiVersion: policy.azure.com/v1alpha1
kind: AzurePolicyDefinition
metadata:
  name: restrict-vm-regions
spec:
  displayName: "Restrict VM deployments to approved regions"
  mode: All
  policyRuleJson: |
    {
      "if": {
        "allOf": [
          { "field": "type", "equals": "Microsoft.Compute/virtualMachines" },
          { "field": "location", "notIn": ["australiaeast", "australiasoutheast"] }
        ]
      },
      "then": { "effect": "Deny" }
    }
  subscriptionId: "00000000-0000-0000-0000-000000000000"

Audit VMs with public IPs (no parameters)

apiVersion: policy.azure.com/v1alpha1
kind: AzurePolicyDefinition
metadata:
  name: audit-public-ip-on-vms
spec:
  displayName: "Audit VMs with public IP addresses"
  mode: Indexed
  policyRule:
    if:
      allOf:
        - field: "type"
          equals: "Microsoft.Network/networkInterfaces"
        - field: "Microsoft.Network/networkInterfaces/ipconfigurations[*].publicIpAddress.id"
          exists: true
    then:
      effect: "Audit"
  managementGroupId: "platform-management-group"

Full Azure resource ID of the created or updated policy definition, populated after the first successful reconcile.

status:
  policyDefinitionId: >-
    /subscriptions/00000000-0000-0000-0000-000000000000
    /providers/Microsoft.Authorization/policyDefinitions/require-https-storage

The spec.version value last successfully written to Azure Policy metadata. Mirrors spec.version after each successful reconcile.

status:
  appliedVersion: "1.0.0"

Standard Kubernetes conditions reflecting the current reconcile state.

Type Status Reason Meaning
Ready True ReconcileSucceeded Definition is in sync with Azure
Ready False ReconcileFailed Last reconcile failed; see message
Ready False AzureAPIError ARM API returned an error 
status:
  conditions:
    - type: Ready
      status: "True"
      reason: ReconcileSucceeded
      message: "Policy definition successfully reconciled"
      lastTransitionTime: "2024-01-15T10:30:00Z"
