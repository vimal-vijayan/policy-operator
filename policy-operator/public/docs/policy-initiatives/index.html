Policy Initiatives

Group related Azure Policy definitions into initiatives (policy sets) using AzurePolicyInitiative.

4 min read

The AzurePolicyInitiative custom resource maps to an Azure Policy Set Definition. An initiative groups one or more policy definitions so they can be assigned and tracked together. The operator reconciles each resource by creating or updating the corresponding policy set definition in Azure.

API version for this resource. Must be policy.azure.com/v1alpha1.

apiVersion: policy.azure.com/v1alpha1

Resource kind. Must be AzurePolicyInitiative.

kind: AzurePolicyInitiative

Standard Kubernetes object metadata.

Unique name of the resource within its namespace.

metadata:
  name: security-baseline-initiative

Kubernetes namespace.

metadata:
  namespace: platform-policies

Map of key/value pairs for organizing resources.

metadata:
  labels:
    team: platform
    env: production

Arbitrary non-identifying metadata.

metadata:
  annotations:
    policy.azure.com/owner: platform-team

Desired state of the AzurePolicyInitiative.

Human-readable name shown in the Azure portal and in compliance reports.

spec:
  displayName: "Security Baseline Initiative"

A human-readable explanation of what the initiative enforces and why.

spec:
  description: "Enforces a security baseline across all subscriptions."

Semantic version of the initiative (e.g. 1.0.0). Written into Azure Policy metadata under the version key; takes precedence over any version key in spec.metadata.

spec:
  version: "1.0.0"

Arbitrary metadata attached to the Azure Policy set definition as a raw JSON object. Common keys: category, version.

spec:
  metadata:
    category: "Security"
    version: "1.0.0"

Declares the parameters accepted by the initiative. Each entry defines a parameter that can be passed through to member policy definitions at assignment time.

The parameter name referenced inside policyDefinitions[].parameters.

spec:
  parameters:
    - name: effect

Data type of the parameter.

Allowed values: String Array Object Boolean Integer Float DateTime 

spec:
  parameters:
    - name: effect
      type: String

Default value used when no value is supplied at assignment time.

spec:
  parameters:
    - name: effect
      type: String
      defaultValue: "Audit"

Restricts the set of values that can be supplied for this parameter.

spec:
  parameters:
    - name: effect
      type: String
      allowedValues: ["Audit", "Deny", "Disabled"]

Azure portal UI hint for this parameter (e.g. location, resourceTypes). Enables portal dropdowns and pickers.

spec:
  parameters:
    - name: allowedLocations
      type: Array
      strongType: "location"

The list of policy definitions included in the initiative. At least one entry is required. Each entry must specify exactly one of policyDefinitionId or policyDefinitionRef.

The full Azure resource ID of a built-in or externally managed custom policy definition. Exactly one of policyDefinitionId or policyDefinitionRef must be specified.

Mutually exclusive with policyDefinitionRef.

spec:
  policyDefinitions:
    - policyDefinitionId: >-
        /providers/Microsoft.Authorization/policyDefinitions/0a914e76-4921-4c19-b460-a2d36003525a

The name of an AzurePolicyDefinition CR in the same namespace. The operator resolves this reference to the Azure resource ID at reconcile time. Exactly one of policyDefinitionId or policyDefinitionRef must be specified.

Mutually exclusive with policyDefinitionId.

spec:
  policyDefinitions:
    - policyDefinitionRef: require-https-storage

Parameter values passed to the referenced policy definition. Keys are parameter names; values follow the Azure Policy parameter value schema.

spec:
  policyDefinitions:
    - policyDefinitionRef: require-https-storage
      parameters:
        effect:
          value: "[parameters('effect')]"

Azure Subscription ID to create the initiative in. When omitted and managementGroupId is not set, the operator's own subscription is used.

spec:
  subscriptionId: "00000000-0000-0000-0000-000000000000"

Management Group ID to create the initiative in. Use this to share the initiative across multiple subscriptions.

spec:
  managementGroupId: "my-management-group"

Observed state of the AzurePolicyInitiative. Populated by the operator after a successful reconcile.

Full Azure resource ID of the created or updated policy set definition.

status:
  initiativeId: >-
    /subscriptions/00000000-0000-0000-0000-000000000000
    /providers/Microsoft.Authorization/policySetDefinitions/security-baseline-initiative

The spec.version value last successfully written to Azure Policy metadata.

status:
  appliedVersion: "1.0.0"

Standard Kubernetes conditions reflecting the current reconcile state (e.g. Ready).

status:
  conditions:
    - type: Ready
      status: "True"
      reason: ReconcileSucceeded
      lastTransitionTime: "2024-01-15T10:30:00Z"

Security baseline with CR references

apiVersion: policy.azure.com/v1alpha1
kind: AzurePolicyInitiative
metadata:
  name: security-baseline-initiative
  namespace: platform-policies
spec:
  displayName: "Security Baseline Initiative"
  description: "Enforces HTTPS on storage accounts and audits VMs with public IPs."
  version: "1.0.0"
  metadata:
    category: "Security"
  parameters:
    - name: storageEffect
      type: String
      defaultValue: "Audit"
      allowedValues: ["Audit", "Deny", "Disabled"]
  policyDefinitions:
    - policyDefinitionRef: require-https-storage
      parameters:
        effect:
          value: "[parameters('storageEffect')]"
    - policyDefinitionRef: audit-public-ip-on-vms
  subscriptionId: "00000000-0000-0000-0000-000000000000"

Built-in policy definitions at management group scope

apiVersion: policy.azure.com/v1alpha1
kind: AzurePolicyInitiative
metadata:
  name: cis-benchmark-initiative
  namespace: platform-policies
spec:
  displayName: "CIS Azure Benchmark (Subset)"
  description: "Applies a subset of CIS Azure Benchmark built-in policies."
  version: "2.0.0"
  metadata:
    category: "Regulatory Compliance"
  policyDefinitions:
    - policyDefinitionId: >-
        /providers/Microsoft.Authorization/policyDefinitions/0a914e76-4921-4c19-b460-a2d36003525a
    - policyDefinitionId: >-
        /providers/Microsoft.Authorization/policyDefinitions/404c3081-a854-4457-ae30-26a93ef643f9
  managementGroupId: "corp-management-group"

Mixed CR and built-in definitions with initiative parameters

apiVersion: policy.azure.com/v1alpha1
kind: AzurePolicyInitiative
metadata:
  name: tagging-and-location-initiative
  namespace: platform-policies
spec:
  displayName: "Tagging and Location Governance"
  description: "Enforces required tags and restricts deployments to approved regions."
  version: "1.2.0"
  metadata:
    category: "Tags"
  parameters:
    - name: tagEffect
      type: String
      defaultValue: "Deny"
      allowedValues: ["Audit", "Deny"]
    - name: allowedLocations
      type: Array
      strongType: "location"
      defaultValue: ["australiaeast", "australiasoutheast"]
  policyDefinitions:
    - policyDefinitionRef: require-cost-center-tag
      parameters:
        effect:
          value: "[parameters('tagEffect')]"
    - policyDefinitionId: >-
        /providers/Microsoft.Authorization/policyDefinitions/e56962a6-4747-49cd-b67b-bf8b01975c4c
      parameters:
        listOfAllowedLocations:
          value: "[parameters('allowedLocations')]"
  managementGroupId: "my-management-group"

Minimal initiative (no parameters)

apiVersion: policy.azure.com/v1alpha1
kind: AzurePolicyInitiative
metadata:
  name: basic-audit-initiative
  namespace: platform-policies
spec:
  displayName: "Basic Audit Initiative"
  policyDefinitions:
    - policyDefinitionRef: audit-public-ip-on-vms
    - policyDefinitionRef: require-https-storage
  subscriptionId: "00000000-0000-0000-0000-000000000000"
